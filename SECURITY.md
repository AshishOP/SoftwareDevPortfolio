# Security Policy

## Supported Versions

Currently being maintained:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within this portfolio template, please follow these steps:

1. **Do NOT** open a public issue
2. Email the maintainer at: ashishshuklacr@gmail.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## Response Timeline

- You will receive an acknowledgment within 48 hours
- We will investigate and provide an update within 5 business days
- Once fixed, we will credit you in the changelog (unless you prefer to remain anonymous)

## Security Best Practices

When forking this project:

- Never commit sensitive data (API keys, tokens, passwords)
- Keep the `.env` file in `.gitignore`
- Regularly update dependencies (if you add any)
- Use HTTPS for all external resources
- Sanitize any user inputs if you add forms
